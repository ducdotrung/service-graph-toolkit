# Graph Incident Triage

Begin read-only: validate/generate the project, identify the affected service
and adjacent manifest edges, then use GitNexus context/impact for implicated
symbols. Use approved health, trace, or log tools only when available; never
ask a model to invent runtime evidence.

Return: observed symptom and time window; likely request/event path; ranked
investigation points with evidence; explicitly missing evidence; safe next
diagnostic command or owner; and containment ideas labelled as suggestions,
not actions. Do not perform remediation, deploys, or data changes through this
skill.
